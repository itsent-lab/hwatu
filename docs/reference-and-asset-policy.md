# 참고 자료 및 자산 사용 정책

이 문서는 FamilyHwatu의 웹앱과 향후 Android, iPhone, iPad, macOS, Windows 네이티브 앱에 공통으로 적용하는 참고 자료·독자 구현·외부 자산 사용 원칙입니다.

## 기본 원칙

FamilyHwatu가 가족용·비상업·비공개 환경에서 사용된다는 사실은 다른 사람이나 회사의 프로그램 내용, 파일 또는 창작물을 허가 없이 가져올 수 있다는 의미가 아닙니다. 공개 서비스 여부와 관계없이 제3자의 권리와 각 자산의 라이선스를 존중합니다.

일반적인 맞고·고스톱 룰, 게임 진행 흐름, 조작 관습과 연출 기법은 공개 게임 플레이 영상을 통해 참고할 수 있습니다. 이때 참고하는 것은 아이디어와 구현 방향이며, 특정 프로그램의 구체적인 코드·데이터·파일이나 고유한 시각·청각 표현을 추출·복제하는 것이 아닙니다.

GitHub 저장소 공개는 FamilyHwatu 서비스를 일반 사용자에게 배포하거나 운영하는 행위와 구분됩니다. 이 저장소는 개발 소스 코드, 테스트, 문서와 재배포 가능한 자산을 공개하며, 실행 중인 서버, 운영 계정·데이터, 실제 운영 설정, 서비스 이용 권한이나 유지보수를 제공하지 않습니다.

## 1. 게임 플레이 영상 참고 범위

개발 과정에서 **피망(Pmang), 한게임(Hangame) 등에서 제공되는 화투 게임의 공개 게임 플레이 영상**을 시청하고 다음과 같은 일반적인 요소를 이해하기 위한 참고 자료로 사용했습니다.

- 맞고·고스톱의 일반적인 진행 흐름과 규칙 표현 방식
- 카드 배치, 정보 표시와 조작 순서 같은 사용자 경험
- 화면 전환, 효과 타이밍, 속도감과 연출 방향
- 음성·효과음·배경음이 사용되는 상황과 전체적인 분위기

참고는 아이디어, 흐름, 사용 경험과 표현 방향을 이해하기 위한 것이며, 피망·한게임 또는 그 운영사와의 제휴, 승인, 후원, 공식 호환이나 대체 서비스 관계를 의미하지 않습니다. 피망과 한게임 등의 명칭은 참고 대상을 사실대로 식별하기 위해서만 사용하며, 각 명칭과 상표의 권리는 해당 권리자에게 있습니다.

## 2. 프로그램과 원본 데이터 비추출

FamilyHwatu는 위 회사들의 프로그램이나 서비스 내부에서 자료를 가져와 만든 프로젝트가 아닙니다.

- 해당 회사 프로그램의 실행 파일, 설치 디렉터리, 패키지·아카이브, 메모리 또는 네트워크 데이터를 열거나 분석하여 콘텐츠를 취득하지 않았습니다.
- 클라이언트를 디컴파일·디스어셈블·역공학·스크래핑하여 소스 코드나 내부 구조를 가져오지 않았습니다.
- 원본 프로그램이나 영상에서 코드, 문구, 화면, 이미지, 애니메이션, 아이콘, 글꼴, 음악, 음성 또는 효과음을 추출·복사·전재·재인코딩하여 프로젝트 파일로 포함하지 않았습니다.
- 특정 회사의 서버와 통신하거나 해당 서비스와 호환되는 비공식 클라이언트를 만들기 위한 프로토콜 분석을 하지 않았습니다.

명시적으로 고지한 화투패 SVG와 오픈소스 의존성을 제외한 소스 코드, 화면 구성, 문구와 자체 자산은 참고 자료를 그대로 가져온 것이 아니라 FamilyHwatu를 위해 새로 작성·설계·제작했습니다.

## 4. 현재 음악과 음성

현재 공개본에는 피망·한게임 또는 다른 게임·영상에서 추출한 녹음 음성, 배경음악, 효과음 파일이 포함되어 있지 않습니다. 대신 공개 재배포가 허용된 다음 자산을 포함합니다.

- 기본 배경음은 공유마당의 CC BY 4.0 「국악 배경음악 #133」을 사용합니다.
- 패 치기·이동·접촉과 나무 타격은 Freesound에서 CC0로 공개한 녹음 효과음을 사용합니다.
- 특수 상황 효과음은 Web Audio API로 실시간 합성하며, 녹음 효과음 로드에 실패해도 합성음이 보조합니다.
- 선언 음성은 Supertonic 3 공개 모델의 F2·M1 음색으로 미리 생성한 한국어 AI 음성 파일을 사용합니다. 음성 파일을 불러오지 못한 경우에만 운영체제나 브라우저의 한국어 TTS를 대체 수단으로 사용합니다.
- 특정 상용 게임·영상의 음원 파일이나 오디오 트랙을 프로젝트 자산으로 복사하지 않습니다.
- 공개 음원의 상세 출처와 파일별 조건은 [`apps/web/public/audio/ATTRIBUTION.txt`](../apps/web/public/audio/ATTRIBUTION.txt)에 기록합니다.

Pixabay의 [「Grace in Hanbok」](https://pixabay.com/music/folk-grace-in-hanbok-321003/)은 앱에 사용할 수 있는 후보지만 원본 파일의 독립 재배포 제한을 고려해 공개 저장소에는 포함하지 않습니다. 별도로 적법하게 취득한 음원은 `apps/web/public/audio/private/`에 두고 `apps/web/.env.local`의 `VITE_GAME_BGM_URL`로 지정하면 로컬과 운영 빌드에서 기본 배경음을 대체할 수 있습니다. 이 디렉터리의 실제 음원은 Git에서 제외되며, 운영자는 해당 음원의 앱 포함·배포 권한과 증빙을 별도로 보관해야 합니다.

### 음성 후보 검토와 공개 판단

- **포함함 — Supertonic 3 생성 음성:** Supertonic 3 모델은 BigScience OpenRAIL-M으로 공개되어 있으며 한국어를 지원합니다. 라이선스 제6조는 모델 제공자가 생성 결과에 권리를 주장하지 않는다고 정하지만, 생성 결과 사용이 라이선스 제한을 위반해서는 안 됩니다. 특히 AI 생성 콘텐츠임을 명확하게 밝혀야 하므로 앱 크레딧, 오디오 고지와 이 문서에 이를 표시합니다. 저장소에는 모델 가중치를 포함하지 않고 생성된 WAV와 파일별 검증 정보만 포함합니다.
- **재배포 가능하지만 미적용 — [Kenney Voiceover Pack](https://opengameart.org/content/voiceover-pack-40-lines):** OpenGameArt에서 CC0로 공개한 전문 게임 선언 음성으로 `Go`, `You win`, `You lose` 등을 포함해 공개 저장소와 앱에서 재배포할 수 있습니다. 그러나 영어 일반 게임 대사라 한국어 화투 선언과 맞지 않아 현재 한국어 게임에는 적용하지 않았습니다. 영어 현지화를 추가할 때 다시 사용할 수 있습니다.
- **포함하지 않음 — 브라우저·운영체제 기본 TTS 추출물:** 런타임 재생은 가능하지만 각 운영체제 음성의 녹음·추출·재배포 조건이 동일하지 않아 생성 파일을 공개 저장소 자산으로 취급하지 않습니다. 공개 음성이 실패할 때 런타임 대체 기능으로만 사용합니다.
- **포함하지 않음 — Azure·Google 등 계정형 음성 서비스 결과:** 앱과 미디어에서 사용할 수 있는 서비스가 있지만 이용 계정, 구독 조건과 생성 시점 약관에 따라 권리 증빙이 달라질 수 있습니다. 이 저장소에서 사용한 계정·약관 증빙이 없으므로 공개 자산으로 포함하지 않습니다.
- **포함하지 않음 — KSS 기반 음성 모델과 불명확한 음성 데이터:** 코드나 모델의 공개 라이선스만으로 학습 음성, 성우 권리와 생성 결과의 공개 재배포 범위가 모두 확인되지는 않습니다. 관련 권리를 추가로 확인하기 전에는 로컬·운영·공개본 모두에 포함하지 않습니다.

Supertonic 3 생성 음성의 대사, 음색, 파일 크기와 SHA-256은 [`apps/web/public/audio/voices/verification.json`](../apps/web/public/audio/voices/verification.json)에 기록하고, 모델 라이선스 전문은 [`licenses/BigScience-OpenRAIL-M.txt`](../licenses/BigScience-OpenRAIL-M.txt)에 포함합니다. 생성 스크립트는 `apps/web/scripts/generate-game-voices.py`이며 `supertonic 1.3.1`, Supertonic 3 모델과 Python용 `numpy`·`soundfile`이 필요합니다. 이 도구들은 일회성 자산 생성에만 사용하며 웹 런타임 의존성에는 포함하지 않습니다.

## 5. 앞으로 추가할 음악·음성·이미지 등의 원칙

향후 다른 게임이나 콘텐츠의 분위기, 장르, 악기 구성, 음성 톤, 대사 속도, 효과가 사용되는 상황 등을 참고할 수 있습니다. 이 경우에도 참고한 원본 파일이나 고유한 표현을 추출·복제하여 사용하지 않고 다음 중 하나의 방식으로만 자산을 추가합니다.

1. **자체 제작**: 직접 작곡·연주·녹음·합성·디자인하거나 프로젝트를 위해 새로 제작합니다.
2. **의뢰 제작**: 제작자와 앱 사용, 수정, 배포 및 필요한 경우 소스 공개 범위를 계약이나 서면으로 확인합니다.
3. **정식 구입 또는 이용 허가**: 판매처나 권리자로부터 해당 플랫폼과 배포 방식에 필요한 라이선스를 구입하거나 명시적인 허가를 받습니다.
4. **공개 라이선스 자산**: 라이선스가 앱 포함과 재배포를 허용하는지 확인하고 저작자 표시, 동일조건변경허락 등 요구사항을 지킵니다.

구입했다는 사실만으로 공개 저장소에 원본 파일을 올릴 권리가 생기는 것은 아닙니다. 앱에 포함할 권리는 있지만 원본 파일의 공개 재배포가 금지된 자산은 Git에 커밋하지 않고, 비공개 빌드·배포 과정에서만 주입해야 합니다. 이런 자산은 MIT License 대상이라고 표시하지 않으며 해당 판매처·권리자의 조건을 따릅니다.

## 6. 자산 추가 전 기록과 공개 고지

음악, 음성, 효과음, 이미지, 아이콘, 글꼴 또는 다른 외부 자산을 추가할 때는 다음 정보를 확인하고 기록합니다.

- 자산명, 제작자·권리자와 공식 출처
- 라이선스 이름, 버전, 원문 링크와 취득일
- 자체 제작·의뢰 제작·구입·공개 라이선스 중 해당하는 취득 방식
- 허용 플랫폼, 앱 포함, 수정, 상업적 이용과 재배포 가능 범위
- 필요한 저작자 표시 문구와 변경 여부
- 프로젝트 내 파일 위치와 적용되는 앱

구매 영수증, 주문번호, 계약서와 계정 정보처럼 공개하면 안 되는 증빙은 저장소 밖에 비공개로 보관합니다. 공개 저장소에는 비밀정보를 제외한 출처와 라이선스 고지만 남기고, 실제 자산을 추가할 때 [`THIRD_PARTY_NOTICES.md`](../THIRD_PARTY_NOTICES.md)와 앱 내 크레딧을 함께 갱신합니다.
