# 네이티브 앱 구현 명세

## 1. 문서 목적

이 문서는 React 웹앱에 현재 구현된 화면, 사용자 조작, 상태 전이, 저장·정산, 오디오와 연출 타이밍을 Android, iOS·iPadOS, macOS와 Windows 네이티브 앱에서 다시 구현하기 위한 기준입니다.

네이티브 앱은 웹 코드를 실행하거나 WebView로 표시하지 않습니다. 각 플랫폼의 네이티브 UI와 저장소를 사용하되, 사용자가 보는 흐름과 같은 입력에서 나오는 게임 결과는 웹과 일치해야 합니다.

이 문서는 2026년 7월 21일의 웹 소스를 조사해 작성했습니다. 아직 웹에 없는 기능을 네이티브 요구사항으로 확대하지 않습니다.

## 2. 기준 자료와 충돌 처리

| 우선순위 | 기준 | 책임 |
| ---: | --- | --- |
| 1 | `shared/contracts/`의 버전이 명시된 계약과 규칙 테스트 벡터 | 기계적으로 검증할 API·데이터·규칙 결과 |
| 2 | [`game-rules.md`](game-rules.md) | 맞고·고스톱 규칙과 정산 공식 |
| 3 | 이 문서와 [`docs/README.md`](README.md)의 네이티브 구현 세트 | 화면 흐름, API, 저장, UI, 오디오, 플랫폼과 출시 기준 |
| 4 | 현재 웹 구현과 웹 테스트 | 문서에 빠진 현재 동작을 확인하는 참고 구현 |
| 5 | 플랫폼별 구현 문서 | 운영체제 API, 안전 영역, 권한과 배포 차이 |

문서와 웹 구현이 다르면 임의로 한쪽을 선택하지 않습니다. 차이를 기록하고 공통 기준을 먼저 수정한 뒤 구현과 테스트를 함께 맞춥니다. 게임 규칙을 이 문서에 중복 정의하지 않고 `game-rules.md`를 참조합니다.

## 3. 현재 제품 범위

현재 네이티브 앱이 재현해야 할 범위는 다음과 같습니다.

- 가족 계정 로그인과 최초 관리자 설정
- 게임 모드 선택
- 2인 맞고와 3인 고스톱 AI 대전
- 점당 게임머니 선택, 게임머니 정산과 0냥 리필
- 맞고 진행 상태 저장·이어하기와 오프라인 로컬 저장
- 고스톱 종료 정산의 재시도와 잔액 스냅숏
- 난이도, 오디오, 자동 치기와 접근성 편의 기능
- 관리자용 가족 계정 추가·중지·재활성화·비밀번호 변경
- 개인정보 처리방침, 자체 코드 라이선스와 제3자 자산 고지

현재 범위가 아닌 항목은 사용자 간 실시간 대전, 매칭, 채팅, 관전, 친구 기능, 결제와 현금 환전입니다. 화면에서 고스톱은 현재 `계획 중` 설명을 사용하지만 실제 로비와 AI 게임 화면에는 진입할 수 있습니다. 이 표시는 네이티브에서 임의로 `완료`로 바꾸지 않습니다.

현재 서버와 API 계약은 쿠키·CSRF 인증을 사용합니다. 네이티브 앱은 운영체제 보호 쿠키 저장소와 [`native-api-auth-contract.md`](native-api-auth-contract.md)의 흐름을 따라야 합니다. 향후 토큰 인증으로 바꾸려면 발급·보관·갱신·폐기와 이전 앱 호환성을 새 계약 버전으로 먼저 확정합니다.

## 4. 전체 화면 흐름

```text
앱 실행
├─ 서버 최초 설정 필요 → 최초 관리자 설정 → 게임 선택
├─ 유효한 세션 있음 → 게임 선택
└─ 세션 없음 → 로그인 → 게임 선택
                         ├─ 맞고 로비
                         │  ├─ 새 판 → 선 정하기 → 시작 → 게임 → 결과
                         │  └─ 이어하기 → 시작 확인 → 게임 → 결과
                         ├─ 고스톱 로비 → 점당 금액 선택 → 게임 → 결과
                         ├─ 가족 회원 관리(관리자만)
                         └─ 출처·개인정보·라이선스
```

웹 경로는 네이티브 앱의 화면 식별자로 사용하며 네이티브 내부에 URL 라우팅을 강제하지 않습니다.

| 화면 식별자 | 웹 경로 | 진입 조건 | 핵심 동작 |
| --- | --- | --- | --- |
| 진입·로그인 | `/`, `/login` | 모두 | 서버 설정 상태와 세션 확인, 로그인 레이어 표시 |
| 최초 관리자 설정 | `/bootstrap` | 사용자 0명, 서버 토큰 설정됨 | 첫 관리자 계정 생성 |
| 게임 선택 | `/home` | 로그인 | 맞고·고스톱 선택, 맞고 저장 턴 표시 |
| 맞고 로비 | `/matgo` | 로그인 | 잔액, 이어하기, 점당 금액별 새 판, 리필 |
| 맞고 게임 | `/matgo/play` | 로그인, 잔액 있음 | 선 정하기, 저장·이어하기, 2인 게임 |
| 고스톱 로비 | `/gostop` | 로그인 | 잔액, 점당 금액별 입장, 미정산 재시도, 리필 |
| 고스톱 게임 | `/gostop/play` | 로그인, 잔액 있음 | 새 3인 AI 게임과 판 종료 정산 |
| 가족 회원 관리 | `/family` | 관리자 | 계정 추가, 상태·비밀번호 변경 |
| 법적 고지 | `/credits`, `/privacy`, `/license` | 모두 | 자산 출처, 개인정보, MIT 라이선스 표시 |

인증이 필요한 화면에서 세션 또는 로컬 프로필을 확보하지 못하면 로그인으로 이동합니다. 알 수 없는 화면 식별자는 진입 화면으로 복귀합니다.

## 5. 공통 화면과 인증

### 5.1 공통 셸

- 상단에는 `가족화투` 브랜드를 표시합니다.
- 로그인 상태에서는 표시 이름, 관리자 전용 `가족 회원`, `로그아웃`을 제공합니다.
- 비로그인 진입 화면에서는 `로그인` 버튼을 제공합니다.
- 하단에는 `화투 패 출처 및 라이선스`, `개인정보 처리방침`, `MIT 라이선스`를 둡니다.
- 로그아웃은 서버 로그아웃을 시도하고 기기 프로필 캐시를 지운 뒤 로그인 화면으로 이동합니다. 서버 요청 실패만으로 로컬 정리를 막지 않습니다.
- 네트워크 데이터를 기다릴 때 빈 화면 대신 브랜드가 있는 로딩 화면과 진행 문구를 표시합니다.

### 5.2 최초 관리자 설정

- 서버 사용자 수가 0명이면 일반 로그인보다 최초 설정을 먼저 표시합니다.
- 서버에 32자 이상의 일회성 초기 설정 토큰이 구성되지 않았으면 입력과 제출을 비활성화하고 설정 안내를 표시합니다.
- 아이디는 한글, 영문 소문자, 숫자와 밑줄로 3~30자입니다.
- 표시 이름은 1~20자, 비밀번호는 15~128자이며 확인 값과 일치해야 합니다.
- 생성 성공 시 로그인 상태로 게임 선택 화면에 진입합니다.

### 5.3 로그인

- 아이디, 비밀번호, `이 기기에서 로그인 유지`를 제공하며 유지 옵션은 기본으로 켭니다.
- 요청 중에는 제출 버튼을 잠그고 중복 요청을 막습니다.
- 네트워크 오류 또는 HTTP 500·502·503·504이면 0.5초, 1초, 2초, 4초, 이후 최대 5초 간격으로 자동 재시도합니다.
- 재시도 중에는 횟수와 `서버 연결 대기 중` 상태를 표시합니다.
- 로그인 창을 닫거나 화면을 벗어나면 진행 중 요청과 재시도 대기를 취소합니다.
- 자격 증명 오류처럼 일시적이지 않은 오류는 즉시 메시지로 표시합니다.

### 5.4 관리자 회원 관리

- 관리자만 진입할 수 있고 일반 회원은 게임 선택 화면으로 돌아갑니다.
- 가족 계정 추가 시 아이디·표시 이름·15자 이상 첫 비밀번호를 입력합니다.
- 목록에는 표시 이름, 아이디, 역할, 최근 로그인, 사용 상태를 표시합니다.
- 계정 중지·재활성화 전에 확인을 받고, 현재 로그인한 자기 계정은 중지할 수 없습니다.
- 비밀번호 변경은 15자 이상의 새 비밀번호를 받아 처리 결과를 알립니다.

## 6. 게임 선택과 로비

### 6.1 게임 선택

- 사용자 표시 이름과 함께 맞고 `2인`, 고스톱 `3인` 카드를 보여줍니다.
- 맞고 서버 저장이 있으면 `저장된 N턴 있음`을 표시합니다.
- 고스톱은 현재 설명 문구가 계획 상태여도 선택하면 고스톱 로비로 이동합니다.

### 6.2 공통 게임머니와 입장 금액

- 점당 금액은 맞고와 고스톱 모두 `100`, `1,000`, `2,000`, `5,000`, `10,000냥`입니다.
- 맞고와 고스톱의 마지막 선택 금액은 서로 다른 기기 설정으로 기억합니다. 기본값은 각각 100냥입니다.
- 사용자 게임머니가 0냥 이하이면 새 게임 진입을 막고 로비에서 `500,000냥 리필 받기`만 제공합니다.
- 리필 중에는 버튼을 잠그고 실패 시 오류를 같은 화면에 표시합니다.
- 결과 정산으로 0냥이 되면 결과 화면에서 다음 판을 시작하지 않고 로비로 이동시켜 리필하도록 합니다.

### 6.3 맞고 로비

- 내 게임머니와 다섯 개 점당 금액의 새 판 입장 버튼을 표시합니다.
- 서버에 저장된 맞고가 있으면 `이어하기 · N턴`을 별도로 표시합니다.
- 새 판 입장은 선택 금액을 저장하고 기존 저장 판 대신 새 게임을 생성합니다.

### 6.4 고스톱 로비

- 내 게임머니와 다섯 개 점당 금액의 3인 방 입장 버튼을 표시합니다.
- 로비 진입 시 사용자별 미정산 요청을 오래된 순서대로 서버에 다시 전송합니다. 하나가 실패하면 뒤 요청을 보내지 않고 다음 진입 때 다시 시도합니다.
- 미정산이 남아 있으면 마지막 로컬 3인 잔액 스냅숏으로 표시 잔액을 유지합니다. 미정산이 없으면 서버 잔액을 새 스냅숏으로 저장합니다.
- 고스톱은 현재 진행 중인 판 이어하기를 제공하지 않습니다.

## 7. 맞고 게임 사용 명세

### 7.1 새 판과 이어하기

- 완전한 새 판의 첫 화면에서는 뒤집힌 12장 중 한 장을 고릅니다. 사람의 패와 컴퓨터의 패를 공개하고 낮 시간(06:00~17:59)에는 높은 월, 밤에는 낮은 월의 패를 고른 쪽을 선으로 정합니다.
- 선 결정 화면에서 점당 금액과 AI 난이도를 함께 바꿀 수 있고, 결정 전에는 나갈 수 있습니다.
- 선택 결과를 1.8초 동안 보여준 뒤 선을 확정합니다.
- 선을 확정한 뒤 `게임 시작`을 눌러 오디오 재생 권한을 열고 패 돌리기를 시작합니다.
- 저장 판은 같은 게임 UUID이면 턴 수가 큰 상태를, 다른 UUID이면 생성 시각이 더 최근인 상태를 선택합니다.
- 선을 이미 정했거나 한 턴 이상 진행된 저장 판은 이어하기 상태로 바로 복원합니다. 선을 정하기 전 저장 판은 선 선택 화면을 유지합니다.

### 7.2 게임판 정보 배치

- 중앙 게임판에는 상대 손패 수, 양쪽 획득패, 바닥패, 더미, 양쪽 점수·고·흔들기·폭탄 상태를 표시합니다.
- 옆 영역에는 상대와 내 표시 이름, 현재 게임머니, 선·현재 차례, 상대 AI 난이도, 미션패와 명령을 표시합니다.
- 내 획득패는 광·열끗·띠·피로 분류하고 국진은 현재 열끗 또는 쌍피 위치에 표시합니다.
- 내 손패는 월 순서를 유지하고 낼 수 있는 패, 보너스패, 흔들기 가능 패와 폭탄 뒤집기 차례를 구분합니다.
- 짧은 가로 화면은 논리 높이 720px을 기준으로 전체 게임판을 비례 축소하며 손패 조작 영역은 별도 하단 영역으로 유지합니다. 일반 태블릿과 세로 화면은 강제 축소하지 않습니다.

### 7.3 기본 조작

- 자기 차례에 손패를 선택하면 손패가 바닥으로 이동하고 더미에서 한 장을 뒤집은 뒤 획득 결과를 반영합니다.
- 같은 월 바닥패가 두 장이면 어느 패를 먹을지 선택창을 표시합니다. 손패로 낸 패와 뒤집은 패에서 각각 선택이 필요할 수 있습니다.
- 폭탄으로 비워 둔 차례에는 손패 대신 더미를 선택합니다.
- 버림패 확인이 꺼져 있으면 마우스·터치·펜 모두 한 번에 냅니다.
- 버림패 확인이 켜져 있으면 터치와 펜은 첫 입력에 선택만 하고 같은 패의 두 번째 입력에 냅니다. 다른 패를 누르면 선택만 이동합니다. 마우스는 계속 한 번에 냅니다.
- 키보드 숫자 `1`~`9`, `0`은 각각 손패 1~10번을 선택합니다. 입력창이 활성화됐거나 선택 대화상자가 열렸을 때는 작동하지 않습니다.

### 7.4 선택 대화상자

| 조건 | 사용자 선택 | 입력 규칙 |
| --- | --- | --- |
| 같은 월 바닥패 2장 | 먹을 바닥패, 또는 다른 손패 | 선택 전 턴 확정 금지 |
| 흔들기 가능 | `흔들기` 또는 `그냥 내기` | 흔들면 세 장 공개 후 선택 패를 냄 |
| 폭탄 가능 | 폭탄 종류별 `한꺼번에 먹기` 또는 `그냥 내기` | 선택 결과에 따라 한 장 또는 묶음 처리 |
| 국진 획득 | `열끗으로 사용` 또는 `쌍피로 사용` | 판 종료 전 획득패에서도 다시 변경 가능 |
| 목표 점수 도달 | `고` 또는 `스톱` | 현재 점수, 다음 고, 스톱 확정 점수·금액 표시 |
| 총통 단독 보유 | `계속` 또는 `스톱` | 4장 흔들기·폭탄 또는 기본 7점 승리 |

컴퓨터가 결정할 때도 같은 고·스톱 패널을 잠금 상태로 보여 사용자가 현재 대기 이유를 알 수 있게 합니다.

### 7.5 편의 기능과 명령

- AI 난이도는 `쉬움`, `보통`, `어려움`, `초고수` 네 단계이며 변경 즉시 현재 저장 상태와 기기 기본값에 반영합니다.
- 점당 금액은 판 종료 전까지 게임판에서 즉시 변경할 수 있고 변경된 값이 이번 판 정산에 적용됩니다.
- `자동 치기`는 내 카드, 폭탄·흔들기, 바닥패, 국진, 총통과 고·스톱까지 자동으로 결정합니다. 다시 누르면 즉시 중지하며 판이 끝나면 자동으로 꺼집니다.
- `무르기`는 사람의 마지막 수를 판당 최대 3회 복원합니다. 자동 치기 중, 연출 중, 선택 대화상자 표시 중, 판 종료 후에는 사용할 수 없습니다.
- `새 판`은 현재 판을 덮어쓸지 확인한 뒤 새 상태를 저장합니다.
- `나가기`는 `예약 나가기`, `바로 나가기`, `계속 게임하기`를 제공합니다. 바로 나가기는 현재 상태 저장을 시도하고 맞고 로비로 이동합니다.
- 예약 나가기는 현재 판을 끝낸 뒤 결과를 보여주고 3초 후 맞고 로비로 이동합니다. 결과 화면의 `지금 나가기`로 즉시 이동할 수도 있습니다.
- 내 프로필 사진은 게임판에서 변경할 수 있습니다. 일반 이미지와 HEIC·HEIF를 허용하고 서버 처리 결과를 반영합니다.

### 7.6 맞고 저장과 동기화

- 기기 저장 키는 사용자별 하나의 진행 판을 가리켜야 하며 상태 버전, 게임 UUID, 생성 시각과 턴 번호를 보존합니다.
- 모든 의미 있는 상태 변경 후 먼저 로컬에 `동기화 대기` 상태로 저장합니다.
- 로그인 상태이고 네트워크가 연결돼 있으면 같은 상태를 서버에 저장한 뒤 로컬 대기 표시를 해제합니다.
- 서버 저장 실패는 현재 게임을 중단시키지 않으며 로컬 저장으로 계속 진행합니다.
- 서버 응답의 사용자·컴퓨터 잔액과 정산 적용 여부를 기준으로 표시 잔액을 갱신합니다.
- 동일 게임 UUID와 종료 정산은 서버에서 중복 적용되지 않아야 합니다.

## 8. 고스톱 게임 사용 명세

### 8.1 방 시작과 화면

- 로비에서 점당 금액을 고르면 사람 1명과 서로 다른 컴퓨터 캐릭터 2명으로 즉시 새 방을 만듭니다.
- 첫 판은 사람이 선이며 이후 승부가 나면 승자가 선, 나가리면 직전 선을 유지합니다.
- 화면에는 세 사람의 이름·게임머니·점수·고·획득패, 각 손패 수, 바닥패, 더미, 현재 차례와 마지막 행동을 표시합니다.
- 상대 손패는 뒷면과 장수만 표시하고 사람 손패만 공개합니다.
- 점당 금액은 방 안에서 바꾸지 않으며 로비에서 선택한 값을 판들이 이어받습니다.

### 8.2 조작과 선택

- 자기 차례에 손패를 선택하고, 같은 월 바닥패가 두 장이면 먹을 패를 고릅니다.
- 뒤집은 패가 두 바닥패와 맞을 때도 뒤집은 패와 후보를 함께 표시합니다.
- 폭탄 빈 차례에는 활성화된 더미 버튼을 눌러 진행합니다.
- 흔들기, 일반 폭탄, 국진, 고·스톱은 맞고와 같은 형태의 큰 선택 패널을 사용합니다.
- 고스톱에는 맞고의 버림패 두 번 터치, 숫자키 단축키와 무르기 기능이 현재 없습니다.
- 자동 치기는 내 행동과 결정을 자동 진행하며 판 종료 시 꺼집니다.
- AI 난이도는 고스톱 전용 기기 설정으로 기억하고 방 안에서도 변경할 수 있습니다.

### 8.3 새 판과 나가기

- 결과 화면의 `한 판 더`는 승자·나가리 결과를 반영해 선과 이월 배수를 정하고 바로 패를 나눕니다.
- 진행 중 `새 판`은 현재 판을 정산하지 않고 즉시 버린 뒤 같은 선, 1배 상태의 새 판을 시작합니다. 현재 웹에는 이 동작의 추가 확인창이 없습니다.
- 게임머니가 0냥이면 다음 판 대신 고스톱 로비 이동만 제공합니다.
- 예약 나가기는 현재 판 종료 후 결과를 보여주고 3초 뒤 고스톱 로비로 이동합니다.
- 바로 나가기는 현재 판을 중단하고 로비로 이동합니다. 고스톱은 진행 판을 이어서 복원하지 않습니다.

### 8.4 정산 신뢰성

- 판이 끝나고 3인 점수 증감이 0이 아니면 로컬 잔액 스냅숏을 먼저 낙관적으로 갱신합니다.
- 서버로 보낼 정산 요청은 사용자별 보류 큐에 게임 UUID로 저장하며 같은 판을 중복 등록하지 않습니다.
- 서버 정산 성공 후에만 보류 큐에서 제거하고 서버가 반환한 세 사람의 잔액으로 스냅숏을 교체합니다.
- 실패한 요청은 유지하고 다음 고스톱 로비 또는 게임 진입 시 재시도합니다.
- 기기 저장을 사용할 수 없어도 현재 판 종료 흐름은 중단하지 않지만, 앱은 사용자에게 동기화 실패 상태를 알 수 있는 진단 정보를 남겨야 합니다.

## 9. 연출 및 입력 타이밍 계약

### 9.1 기본 원칙

- 게임 규칙 상태가 타이머에 의존해서 달라지면 안 됩니다. 타이머는 표시와 입력 잠금에만 사용합니다.
- 한 턴의 카드 이동과 필수 선택이 끝나기 전에 다음 플레이어 입력을 허용하지 않습니다.
- 대화상자에서 사용자의 결정이 필요한 시간에는 자동으로 임의 선택하지 않습니다. 자동 치기가 켜진 경우에만 자동 전략이 대신 결정합니다.
- 앱이 백그라운드로 가면 AI·자동 치기 예약과 오디오를 일시 중지하거나 취소하고, 복귀 시 현재 상태에서 한 번만 다시 예약합니다.
- 화면 종료 시 남아 있는 타이머, 애니메이션 콜백과 오디오 예약을 취소합니다.
- 느린 기기에서 애니메이션이 지연돼도 동일 행동이 두 번 적용되지 않도록 상태 전이에 게임 UUID·턴 번호 또는 단발성 실행 표식을 사용합니다.

### 9.2 현재 웹 기준 시간표

| 사건 | 기준 시간 | 사용자에게 보이는 순서와 입력 상태 |
| --- | ---: | --- |
| 맞고 선 선택 카드 뒤집기 | 620ms 전환, 결과 1,800ms 유지 | 선택 즉시 다른 카드 잠금 → 두 패 공개 → 1.8초 후 선 확정 |
| 맞고 패 돌리기 | 총 1,700ms | 20장, 장당 시작 간격 43ms, 개별 이동 720ms; 완료 전 게임 입력 잠금 |
| 고스톱 패 돌리기 | 입력 잠금 1,550ms | 21장, 세 좌석 순환, 장당 시작 간격 48ms, 개별 이동 680ms |
| 맞고 사람 손패 이동 | 320ms | 손패 입력 잠금 → 카드가 바닥 또는 획득 영역으로 이동 |
| 맞고 더미 뒤집기 | 260ms | 손패 이동 뒤 실행; 두 바닥패 선택이 필요하면 이동 뒤 선택창 유지 |
| 맞고 보너스 대체패 | 240ms | 보너스 획득 뒤 더미에서 손패로 보충 |

| 고스톱 손패 이동 | 280ms | 사람·컴퓨터 공통; 이동 중 다음 입력 잠금 |

| 이동 목적지 강조 | 170ms | 카드 도착 직후 밝기·크기 강조, 게임 진행을 추가로 막지 않음 |
| 맞고 자동 치기 사람 대기 | 1,200ms | 선택창·선언·연출이 없고 내 차례일 때만 예약 |
| 고스톱 자동 치기 사람 대기 | 1,300ms | 현재 상태가 계속 내 차례일 때 한 번 실행 |

| 판 종료 결과 공개 | 맞고 2,200ms | 승리·패배·나가리 음성과 종료 선언을 먼저 보여준 뒤 결과창 표시 |
| 고스톱 판 종료 결과 공개 | 즉시 | 종료 상태와 결과창을 즉시 반영하고 종료 선언을 그 위에 함께 표시 |
| 예약 나가기 | 결과 표시 후 3,000ms | 결과 안내와 `지금 나가기` 제공 후 자동으로 로비 이동 |

카드 이동의 기준 곡선은 빠르게 출발해 도착 전에 감속하는 `cubic-bezier(0.2, 0.75, 0.18, 1)`입니다. 네이티브 프레임워크에서 동일 곡선을 지원하지 않으면 시각적으로 가장 가까운 ease-out 곡선을 사용하고 차이를 기록합니다.

### 9.3 맞고 AI 생각 시간

일반 자동 치기가 아닐 때 AI 생각 시간은 난이도와 판의 고정 난수값으로 결정하므로 같은 판·같은 턴에서는 재현 가능해야 합니다.

| 난이도 | 일반 빠른 판단 범위 |
| --- | ---: |
| 쉬움 | 650~1,050ms |
| 보통 | 800~1,350ms |
| 어려움 | 950~1,600ms |
| 초고수 | 1,050~1,800ms |

- 고·스톱, 총통 또는 복잡한 수는 `2,200 + 변동값 × 2,800ms`, 최대 5,000ms를 사용합니다.
- 일반 수라도 고정 변동값이 0.72를 넘으면 긴 생각 시간을 사용합니다.
- 자동 치기 중 AI 표시는 900~1,300ms로 단축합니다.
- 남은 초 숫자는 표시하지 않고 문구와 진행 막대만 표시합니다.
- 생각 시간 동안 게임판 입력을 막고, 차례·게임 UUID·턴이 바뀌면 예약 행동을 폐기합니다.

### 9.4 선언과 효과 시간

아래 시간은 선언 상태의 생명주기와 다음 선언 예약에 사용하는 기준입니다. 일반 선언 레이어 자체의 기본 등장·퇴장 애니메이션은 950ms이고, 5고 이상과 삼연뻑처럼 별도 스타일이 있는 효과는 1,750~1,800ms까지 보입니다.

| 선언·효과 | 생명주기 기준 |
| --- | ---: |
| 일반 `짝` 획득 | 620ms |
| 첫 뻑 | 950ms |
| 연속뻑 | 1,350ms |
| 삼연뻑 | 1,750ms |
| 흔들기 | 1,100~1,200ms |
| 폭탄·쪽·따닥·자뻑·싼 패 먹기 | 1,100~1,200ms |
| 고스톱 싹쓸이 | 1,350ms |
| 보너스 쌍피·삼피 | 1,250ms |
| 점수 족보 | 1,250ms |
| 미션 획득 | 1,100ms |
| 1~4고 | 1,150ms |
| 5고 이상 축포 | 1,800ms |
| 스톱·최종 정산 | 1,600ms |
| 나가리 | 1,500ms |

- 한 행동에 보너스와 다른 특수 효과가 겹치면 보너스를 먼저 표시하고 다음 선언을 약 1,120ms 뒤 시작합니다.
- 특수 상황으로 쌍피·삼피를 가져오면 특수 선언과 피 이동 안내를 함께 표시하고 피 이전 음향은 선언 시작 약 120ms 뒤 재생합니다.
- 컴퓨터의 일반 획득·점수 음향은 상태 변화 뒤 약 190ms, 폭탄 빈 차례의 획득 음향은 약 150ms 지연될 수 있습니다.
- 선언은 게임판을 가리더라도 자체적으로 포인터 입력을 받지 않습니다. 다만 턴 처리 중 `busy` 상태와 선택 패널은 중복 게임 입력을 막아야 합니다.

### 9.5 오디오 순서

- 첫 사용자 입력에서 오디오 세션을 활성화합니다. 활성화 실패가 게임 진행을 막으면 안 됩니다.
- 카드 내기 음향은 손패 이동 시작, 뒤집기 음향은 더미 이동과 함께 재생합니다.
- 결정이 필요한 패널을 열 때 결정 음향, 취소·그냥 내기에는 취소 음향을 재생합니다.
- 선언 음성은 해당 상태 변화와 중앙 선언이 나타나는 시점에 맞춥니다.
- 승리·패배·나가리 음성을 결과창보다 먼저 재생하고 맞고 결과창은 2.2초 뒤 표시합니다.
- 정산 게임머니 음향은 서버가 이번 요청에서 정산을 실제 적용했고 결과창이 표시됐을 때 한 번만 재생합니다.
- 기본 설정은 음소거 해제, 음량 28%, 배경음 끔입니다. 음소거·0~100% 음량·배경음 켜기 상태를 기기별로 기억합니다.

배경음을 켜면 판 시작 시 차분한 단계로 초기화하고 사건이 커질 때 같은 판 안에서만 더 높은 단계로 올립니다. 스톱·승리·패배·나가리는 결과 단계로 전환합니다.

| 배경음 단계 | 재생 속도 | 배경음 게인 | 전환 기준 시간 |
| --- | ---: | ---: | ---: |
| 차분함 | 1.000 | 0.130 | 0.90초 |
| 진행 | 1.025 | 0.140 | 0.65초 |
| 긴장 | 1.060 | 0.152 | 0.45초 |
| 절정 | 1.095 | 0.162 | 0.28초 |
| 결과 | 0.960 | 0.115 | 0.75초 |

점수는 5점부터 긴장, 10점부터 절정 단계로 올리고 고는 1~2고에서 긴장, 3고부터 절정 단계로 올립니다. 낮은 단계의 사건이 뒤이어 발생해도 판 도중 배경음 단계를 낮추지 않습니다.

### 9.6 동작 줄이기와 접근성

- 운영체제의 동작 줄이기 설정이 켜지면 반복·이동·폭발 애니메이션은 즉시 완료에 가깝게 줄입니다.
- 동작을 줄여도 AI 판단 대기, 게임 상태 변화, 선택 대화상자, 결과 정보와 오디오 설정을 생략하지 않습니다. 불필요한 연출 대기는 제거할 수 있습니다.
- 상태 안내와 선언은 화면 읽기용 live region 또는 플랫폼의 동등한 접근성 알림으로 한 번만 전달합니다.
- 모달은 제목부터 논리적인 포커스 순서를 제공하고 닫힐 때 원래 조작 위치로 포커스를 돌립니다.
- 카드 선택은 색상만으로 구분하지 않고 테두리·위치·설명과 접근성 레이블을 함께 제공합니다.
- 글자 확대 시 핵심 점수, 선택 버튼과 손패가 잘리면 안 됩니다. 장식보다 조작 가능 영역을 우선합니다.

## 10. API와 기기 저장 경계

### 10.1 현재 서버 API

| 영역 | 메서드·경로 | 사용 목적 |
| --- | --- | --- |
| 설정 | `GET /api/auth/status` | 최초 관리자 필요 여부와 토큰 활성 여부 |
| CSRF | `GET /api/auth/csrf` | 쓰기 요청 토큰 |
| 인증 | `POST /api/auth/bootstrap`, `/login`, `/logout` | 최초 관리자, 로그인, 로그아웃 |
| 세션 | `GET /api/session` | 사용자, CSRF 토큰, 기기 ID |
| 대시보드 | `GET /api/dashboard` | 사용자 잔액, 맞고 저장 요약, 당일 기록 |
| 잔액 | `POST /api/balance/refill` | 사용자 게임머니 500,000냥 리필 |
| 프로필 | `GET /api/profile/image/{userId}/{version}`, `PUT /api/profile/image` | 프로필 이미지 조회·변경 |
| 회원 관리 | `GET·POST /api/users`, `PATCH /api/users/{id}/status`, `PUT /api/users/{id}/password` | 관리자 회원 관리 |
| 맞고 | `GET·PUT /api/games/matgo` | 진행 상태 조회·저장과 종료 정산 |
| 고스톱 | `POST /api/games/gostop/settle` | 종료된 판의 3인 잔액 정산 |
| 호환성 | `GET /api/client/status` | 클라이언트·서버 버전 호환 확인 |

모든 응답은 성공 여부, 데이터와 오류 코드·메시지를 담은 공통 봉투 형식을 사용합니다. 쓰기 요청은 현재 CSRF 토큰이 필요하며 419 또는 `CSRF_FAILED`이면 토큰을 한 번 새로 받아 재시도합니다.

`shared/contracts/client-api-v1.json`은 현재 macOS 헤더와 쿠키·CSRF 인증만 명시합니다. Android, iOS·iPadOS와 Windows 클라이언트 값 및 최소 버전 필드는 각 플랫폼을 시작하기 전에 공통 계약을 확장해야 합니다.

### 10.2 기기 저장 대응

| 웹 저장 | 의미 | 네이티브 대응 |
| --- | --- | --- |
| IndexedDB 프로필 | 마지막 로그인 사용자 캐시 | 암호화가 필요한 값과 일반 캐시를 분리한 플랫폼 저장소 |
| IndexedDB 맞고 상태 | 사용자별 진행 판과 동기화 대기 | 원자적 로컬 DB 레코드 |
| localStorage 환경 설정 | 난이도, 점당 금액, 버림패 확인, 오디오 | 플랫폼 사용자 설정 저장소 |
| localStorage 고스톱 보류 큐 | 사용자별 미정산 요청 | 중복 키가 있는 내구성 큐 |
| localStorage 고스톱 잔액 | 정산 직후 세 사람 잔액 스냅숏 | 사용자별 원자적 스냅숏 |
| sessionStorage 상대 배정 | 한 로비 입장 세션의 상대 이름·음성 | 게임 세션 수명 상태 |

비밀번호, 초기 설정 토큰과 향후 네이티브 인증 토큰을 일반 설정 저장소에 기록하지 않습니다. 인증 비밀은 Keychain, Android Keystore 계열, Windows 자격 증명 저장소처럼 운영체제가 제공하는 보안 저장소를 사용합니다.

## 11. 네이티브 구현 구조 기준

각 플랫폼은 다음 책임을 분리합니다.

- 화면: 화면 배치, 접근성, 애니메이션과 사용자 입력
- 화면 상태: 로딩·오류·선택창·진행·결과 상태와 단발성 이벤트
- 게임 도메인: 카드, 턴, 규칙, 점수, AI와 정산
- 저장소: 프로필, 환경 설정, 맞고 판, 고스톱 정산 큐와 마이그레이션
- API: 요청·응답, 인증, 오류 코드, 재시도와 호환성 확인
- 오디오: 공통 파일 매핑, 재생 순서, 음량과 생명주기

웹 TypeScript 구현이나 다른 플랫폼의 UI·저장소 코드를 직접 참조하지 않습니다. 언어 사이에서 공유할 것은 `shared/contracts/`의 JSON 계약과 테스트 벡터뿐입니다.

## 12. 구현 및 검증 순서

1. 대상 플랫폼의 인증 방식과 `client-api` 호환성 계약을 확정합니다.
2. `game-rules.md`를 기반으로 규칙 엔진을 구현하고 공통 JSON 테스트 벡터를 통과시킵니다.
3. 프로필·환경 설정·맞고 저장·고스톱 정산 큐의 버전과 마이그레이션을 구현합니다.
4. 진입, 로그인, 게임 선택과 로비를 구현합니다.
5. 맞고의 선 정하기, 기본 턴, 선택 패널, 저장·이어하기와 결과를 구현합니다.
6. 고스톱의 3인 턴, 선택 패널, 정산 큐와 결과를 구현합니다.
7. 이 문서의 시간표에 맞춰 카드 이동, AI 대기, 선언, 오디오와 입력 잠금을 연결합니다.
8. 웹과 같은 고정 난수·카드 상태를 사용해 규칙 결과, 상태 순서와 스크린샷을 비교합니다.

- [ ] 패 돌리기 중 게임 입력이 적용되지 않습니다.

- [ ] AI 대기 중 차례가 바뀌거나 화면을 나가면 예약 행동이 취소됩니다.
- [ ] 자동 치기를 끄면 아직 실행되지 않은 사람 자동 행동이 취소됩니다.
- [ ] 결과 정산과 게임머니 음향이 앱 재활성화나 재렌더링으로 중복되지 않습니다.
- [ ] 예약 나가기가 결과 표시 후 3초에 동작하고 수동 나가기도 가능합니다.
- [ ] 백그라운드·포그라운드 전환 뒤 타이머와 오디오가 중복 실행되지 않습니다.
- [ ] 동작 줄이기에서 연출은 축소되지만 선택과 결과 정보는 유지됩니다.
