# 네이티브 오디오·효과 이벤트 계약

## 1. 목적

플랫폼마다 재생 API가 달라도 같은 게임 사건에 같은 계열의 효과음·음성과 시각 선언을 한 번만 발생시키기 위한 기준입니다. 파일 경로와 실제 자산 목록은 `apps/web/src/lib/audioAssets.ts`, 해시 검증은 각 `apps/web/public/audio/**/verification.json`, 라이선스는 `apps/web/public/audio/ATTRIBUTION.txt`가 기준입니다.

## 2. 설정과 생명주기

기본 설정은 음소거 꺼짐, 마스터 음량 `0.28`, BGM 꺼짐입니다. 효과·음성·BGM은 마스터 음량과 음소거를 모두 따릅니다.

- 첫 명시적 사용자 입력 후 오디오 세션을 활성화합니다.
- 앱이 비활성·백그라운드가 되면 BGM과 예약 음성을 일시 중지하거나 취소합니다.
- 화면을 벗어나면 해당 판의 예약 음성·효과 콜백을 취소합니다.
- 통화·다른 앱 오디오 정책은 플랫폼 관례를 따르며 독점 재생을 강제하지 않습니다.
- 무음 모드 정책은 게임 효과의 성격과 스토어 가이드에 맞춰 플랫폼별로 검증하되, 앱 내 음소거가 최우선입니다.

## 3. 사건 매핑

| 게임 사건 | 효과 계열 | 음성/표시 |
| --- | --- | --- |
| 패 돌리기·내기·뒤집기·획득 | deal, cardDeal, cardSlide, cardContact | 획득 음성은 카드 접촉 뒤 |
| 선택 열기·선택·취소 | decision, select, cancel | 화면 선택 상태와 같은 프레임에 시작 |
| 보너스 2·3피 | bonusTwo/Three | 행위자별 음성, 연속 획득은 한 번으로 묶을 수 있음 |
| 피 2·3장 이동 | peeTransferTwo/Three | 이동 연출과 동기화 |
| 쪽·따닥·쓸·뻑·자뻑·뻑먹기 | 동일 이름 특수 효과 | 행위자별 특수 음성과 선언 표시 |
| 폭탄·흔들기·총통 | bomb, shake, chongtong | 결정 적용 뒤 한 번 |
| 국진 선택 | gookjinAnimal/Double | 선택된 역할을 읽음 |
| 미션·점수 | mission, score | 실제 점수 상태 반영 뒤 |
| 고·스톱 | go/goHigh, stop | 고 횟수와 행위자 음성 |
| 승·패·나가리 | win, lose, nagari | 결과 상태 확정 뒤 |
| 게임머니 이동 | moneyWin/MoneyLose | 잔액 이동 연출 시작 때 |
| 시작·선 결정·무르기·자동 치기 | start, dealer, undo, autoplay | 명령 성공 뒤 |

규칙 함수가 낸 사건 ID를 오디오 계층이 소비하게 하고, 화면 컴포넌트마다 규칙을 다시 추론하지 않습니다. 재렌더링·화면 복원만으로 소리를 재생하면 안 됩니다.

## 4. 현재 음성 시작 지연

지연은 효과음의 충격음을 가리지 않도록 음성 시작만 늦추는 값입니다. 게임 상태 전이는 이 타이머를 기다리지 않습니다.

| 사건 | 지연 |
| --- | ---: |
| 일반 획득 | 80ms |
| 보너스피 | 150ms |
| 피 이동 | 140ms |
| 뻑·폭탄 | 120ms |
| 흔들기 | 사람 140ms, 상대 170ms |
| 총통 | 130ms |
| 선 결정 | 90ms |
| 국진 | 100ms |
| 무르기 | 70ms |
| 쪽·따닥·쓸 등 특수 수 | 110ms |
| 미션 | 170ms |
| 점수 | 140ms |
| 고 | 90ms, 5고 이상 220ms |
| 스톱 | 70ms |
| 승리·패배·나가리 | 320ms·300ms·230ms |
| 게임 시작 | 140ms |

전체 카드 이동·AI·결과 표시 시간은 [`native-app-implementation-spec.md`](native-app-implementation-spec.md)의 타이밍 표를 따릅니다.

## 5. 중복과 우선순위

- 한 턴의 같은 사건은 `gameUuid + turnNumber + eventKind + sequence`로 한 번만 소비합니다.
- 여러 특수 수가 동시에 발생하면 가장 강한 선언을 먼저 재생하고, 보조 피 이동은 짧게 뒤따릅니다.
- 승패 음성은 점수 음성을 중단할 수 있으며, 화면을 떠난 판의 음성은 재생하지 않습니다.
- 짧은 시간에 음성이 겹치면 이전 일반 획득 음성을 중단하거나 생략할 수 있지만 고·스톱·승패 같은 결정 음성은 보존합니다.
- BGM은 효과·음성 재생 중 자동으로 낮아질 수 있으나 효과음 자체의 상대 크기는 유지합니다.

## 6. 자산 패키징과 검증

- 자산 파일명을 바꾸지 말고 플랫폼 번들 매핑표를 둡니다.
- 배포 전에 verification JSON의 경로·바이트 크기·해시와 패키지 자산을 비교합니다.
- 누락 자산은 앱 시작 실패가 아니라 해당 소리 생략으로 처리하되 배포 검증은 실패시킵니다.
- 음성 캐릭터 매핑이 없는 경우 임의 캐릭터 음성을 섞지 않고 공통 또는 무음 대체를 사용합니다.
- BGM 파일의 출처·라이선스 고지는 앱 크레딧과 스토어 자료에 반영합니다.

## 7. 접근성·테스트

- 음성이 없어도 선언 텍스트와 결과가 동일하게 이해돼야 합니다.
- 동작 줄이기는 시각 효과를 축소하지만 단발 효과음을 자동으로 켜지거나 끄지 않습니다.
- 단위 테스트는 사건→자산 키와 중복 제거를 검증하고 실제 스피커를 사용하지 않습니다.
- 통합 테스트는 배경/복귀, 화면 이탈, 빠른 새 판, 음소거 전환, Bluetooth·전화 중단 뒤 재생 상태를 확인합니다.
