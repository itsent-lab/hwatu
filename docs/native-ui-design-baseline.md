# 네이티브 UI·디자인·접근성 기준

## 1. 목적

웹 픽셀을 기계적으로 복사하지 않고, 플랫폼 네이티브 UI로 같은 정보 위계와 조작 결과를 제공하기 위한 기준입니다. 화면별 동작은 [`native-app-implementation-spec.md`](native-app-implementation-spec.md), 여기서는 디자인 토큰, 반응형 기준, 접근성과 기준 화면 관리만 다룹니다.

## 2. 공통 시각 토큰

현재 웹의 핵심 색을 초기 네이티브 토큰으로 사용합니다. 접근성 대비가 부족하면 의미는 유지하면서 명도 조정 토큰을 별도로 둡니다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `red` | `#B9322B` | 주요 강조·위험 |
| `deepRed` | `#7C1719` | 강조의 어두운 면 |
| `ink` | `#201713` | 기본 글자 |
| `paper` | `#FFFAF0` | 종이색 표면 |
| `gold` | `#D9A536` | 점수·보상 강조 |
| `green` | `#153D2B` | 게임 테이블·성공 계열 |
| `line` | `rgba(70,44,27,0.14)` | 구분선 |
| `shadow` | `0 18 50 rgba(73,45,28,0.13)` | 큰 카드 표면 기준 |

기본 글꼴 우선순위는 플랫폼 시스템 한글 글꼴이며, 웹 기준은 Apple system, Apple SD Gothic Neo, Noto Sans KR입니다. 숫자·금액은 자리 폭이 흔들리지 않는 tabular 숫자를 우선합니다.

화투 카드 원본 비율은 `103.2:168.2`입니다. 카드를 자르거나 임의로 늘이지 않고 contain 방식으로 표시합니다.

## 3. 정보 위계와 컴포넌트 상태

모든 버튼·카드·선택 항목은 최소 `default`, `pressed`, `focused`, `selected`, `disabled`, `loading` 상태를 설계합니다. 게임 카드는 추가로 `playable`, `notPlayable`, `touchPrimed`, `shakeCandidate`, `bombCandidate`, `moving`을 구분합니다.

- 한 화면의 주 행동은 하나만 가장 강하게 표시합니다.
- 선택창은 선택 이유, 후보, 취소 가능 여부를 함께 표시합니다.
- 입력 잠금 중에는 누를 수 없는 이유를 현재 행동 문구로 보여줍니다.
- 게임머니·점수·고 횟수는 색만으로 구분하지 않고 라벨과 숫자를 함께 둡니다.
- 오류는 해당 입력 근처에 표시하고, 전체 화면 오류는 재시도 또는 돌아가기 행동을 제공합니다.
- 로딩 스피너만 두지 말고 `게임 저장을 확인하는 중`처럼 작업을 설명합니다.

## 4. 반응형과 안전 영역

웹 검증 구간은 폭 760, 720, 640, 520, 470과 가로 높이 650·620입니다. 네이티브는 픽셀 분기 대신 compact/regular 크기 클래스와 실제 안전 영역을 사용하되 다음 결과를 만족해야 합니다.

- 최소 유효 폭 320pt에서도 로그인·로비의 주 행동과 오류 문구가 잘리지 않습니다.
- 세로 compact 폭에서는 보조 정보가 아래로 흐르고 수평 스크롤을 만들지 않습니다.
- 짧은 가로 게임판은 논리 높이 720 기준으로 비례 축소할 수 있지만 손패 터치 영역은 별도 하단 영역에서 최소 크기를 유지합니다.
- 노치, Dynamic Island, 홈 표시기, 데스크톱 제목 표시줄과 Windows 배율을 침범하지 않습니다.
- 키보드가 열려도 활성 입력과 제출 오류가 보입니다.
- 창 크기 변경 중 게임 규칙 상태나 선택이 초기화되지 않습니다.

권장 최소 터치 목표는 iOS 계열 44×44pt, Android 48×48dp입니다. 데스크톱에서도 카드 포인터 영역은 시각 카드보다 작아지지 않게 합니다.

## 5. 입력 장치

| 입력 | 기준 |
| --- | --- |
| 터치·펜 | 맞고 버림 확인이 켜지면 첫 탭 선택, 같은 카드 두 번째 탭 실행 |
| 마우스 | 카드 한 번 클릭으로 실행; hover는 보조 힌트일 뿐 필수 정보 금지 |
| 키보드 | 논리적 Tab 순서, Enter/Space 활성화, 맞고 손패 1~9·0 단축키 |
| 게임패드 | 현재 필수 범위 아님; 추가 시 포커스와 취소 규칙을 문서 버전으로 확정 |

대화상자가 열리면 포커스를 내부에 가두고 닫을 때 원래 조작으로 돌려보냅니다. 파괴·포기 행동은 기본 포커스로 두지 않습니다.

## 6. 접근성

- 본문과 핵심 숫자는 WCAG AA 대비를 목표로 합니다.
- 운영체제 글자 크기 확대에서 인증·관리 화면은 재배치되고 잘리지 않아야 합니다.
- 카드 접근성 이름은 `3월 광`, `9월 열끗 국진`처럼 월·종류·특성을 포함합니다.
- 상대의 숨은 패는 개별 내용을 읽지 않고 `상대 손패 7장`으로 알립니다.
- 점수 변화, 차례 변경, 선택 필요, 정산 완료를 접근성 알림으로 한 번만 전달합니다.
- `동작 줄이기`에서는 장거리 비행·번쩍임·반복 축하를 축소하거나 즉시 상태 전환으로 대체합니다. 규칙 입력 잠금 시간은 필요한 최소만 유지합니다.
- 색각 차이를 위해 선·현재 차례·사용 가능 상태를 테두리, 아이콘, 텍스트로 중복 표현합니다.
- 효과음이 꺼져도 모든 중요한 사건은 시각·접근성 텍스트로 확인할 수 있어야 합니다.

## 7. 기준 화면 세트

스크린샷은 디자인의 유일한 계약이 아니라 회귀 비교 자료입니다. 각 플랫폼은 아래 상태를 밝은 모드 기준으로 캡처하고, 지원하는 경우 큰 글자·동작 줄이기·고대비 변형을 추가합니다.

| 그룹 | 필수 상태 |
| --- | --- |
| 인증 | 최초 설정 가능/불가, 로그인 기본/오류/재시도 |
| 홈·로비 | 일반 잔액, 0냥 리필, 맞고 이어하기, 고스톱 미정산 |
| 맞고 | 선 선택 전/결과, 패 돌리기, 일반 턴, 바닥 2장 선택, 흔들기, 폭탄, 국진, 고·스톱, 승/패/나가리 |
| 고스톱 | 일반 턴, 3인 획득패, 선택창, 고·스톱, 승자별 결과, 나가리 |
| 관리·법적 | 회원 목록, 생성 오류, 상태 확인창, 크레딧·개인정보·라이선스 |
| 시스템 | 오프라인, 동기화 대기, 세션 만료, 호환 불가, 저장 손상 |

파일명은 `<platform>-<size>-<screen>-<state>-<locale>.png` 형식으로 기록합니다. 예: `ios-compact-matgo-floor-choice-ko.png`. 캡처에는 실제 계정·서버 주소·토큰을 넣지 않습니다.

## 8. 시각 회귀 판정

- 카드 누락, 겹침으로 인한 정보 손실, 잘린 행동, 잘못된 안전 영역은 실패입니다.
- 안티앨리어싱·운영체제 기본 컨트롤의 미세 차이는 픽셀 완전 일치 대상이 아닙니다.
- 화면 차이가 의도된 플랫폼 관례라면 [`native-platform-release.md`](native-platform-release.md)에 이유를 기록합니다.
- 기준 이미지를 갱신할 때는 코드 변경과 함께 이전·이후를 검토하며, 실패를 통과시키기 위해 무조건 기준을 교체하지 않습니다.
