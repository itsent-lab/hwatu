# 보안 정책

FamilyHwatu 저장소는 개발 소스 코드를 공개하며 NSJ가 운영하는 공개 서비스나 관리형 호스팅을 제공하지 않습니다. 보안 업데이트, 지속적인 유지보수 또는 특정 응답 시간을 보장하지 않지만, 저장소 자체에서 확인되는 보안 문제는 책임 있게 다룹니다.

## 취약점 제보

공개 이슈에 실제 비밀번호, 토큰, 개인정보, 공격 절차 또는 운영 서버 정보를 올리지 마십시오. 보안상 민감한 문제는 `itsent@itsent.co.kr`로 재현에 필요한 최소한의 정보만 보내 주세요. 메일을 보냈다는 사실만으로 수정, 배포 또는 응답 기한이 약속되는 것은 아닙니다.

## 공개 저장소에 포함하지 않는 정보

- 실제 DB 연결 문자열, 계정과 비밀번호
- 초기 관리자 설정 토큰, API 토큰과 기타 인증정보
- 인증서, 개인키, 서명 키와 네이티브 앱 배포 자격증명
- 실제 도메인, 서버 사용자명, 내부 설치 경로가 들어간 운영 설정
- 운영 로그, 백업, 업로드 파일, 가족 계정과 게임 데이터

공개 예제의 빈 값과 예시 값은 실제 운영 값으로 바꾼 뒤 추적되는 파일에 커밋하지 않습니다. 비밀정보는 환경 변수나 운영 환경의 비밀 저장소에 두고, 공개 전에 현재 파일뿐 아니라 Git 전체 이력도 검사해야 합니다.

## 최초 관리자 생성

사용자가 없는 새 DB에서는 `Bootstrap:Token`에 32자 이상의 임의 토큰이 설정된 경우에만 첫 관리자를 만들 수 있습니다. 토큰은 비밀로 전달하고 HTTPS에서 한 번만 사용한 다음 설정에서 제거하고 서버를 재시작합니다. 애플리케이션은 동시 생성 요청을 DB 잠금으로 직렬화하지만, 외부 네트워크에 노출하기 전에 관리자를 먼저 만드는 방식이 가장 안전합니다.

## 운영 보안 경계

- 운영 환경은 HTTPS, 방화벽, 신뢰할 수 있는 리버스 프록시, 제한된 DB 권한, 백업과 로그 접근 통제를 별도로 구성해야 합니다.
- 이 프로젝트는 가족 구성원이 사용하는 신뢰된 소규모 환경을 전제로 합니다. 게임 상태와 정산 요청은 서버에서 크기·범위·중복을 검사하지만 전체 게임을 서버가 재연산하는 경쟁형 권위 서버 구조는 아닙니다.
- 가상 게임머니는 현실의 금전이나 환전 가능한 자산이 아닙니다. 불특정 사용자, 경쟁 서비스, 현금성 보상 또는 규제 대상 운영에 그대로 사용해서는 안 됩니다.
- 외부 공개 서비스로 확장하려면 서버 권위형 게임 검증, 감사 로그, 계정 복구, MFA, 침입 탐지, 개인정보 처리와 법적 요구사항을 별도로 설계하고 보안 검토를 받아야 합니다.

## 의존성과 배포물

소스 공개 전에 JavaScript와 .NET의 직접·전이 의존성 취약점을 검사합니다. 앱이나 서버 바이너리를 배포할 때는 그 배포물에 실제 포함된 모든 제3자 구성요소의 라이선스와 NOTICE도 함께 확인해야 합니다.
