# Hwatu

React와 ASP.NET Core로 만든 한국 화투 웹게임입니다. 어르신과 가족이 쉽고 편안하게 즐길 수 있는 건강한 여가 환경을 목표로 제작했으며, 2인 맞고와 3인 고스톱, AI 상대, 게임 저장 및 사용자 관리 기능을 포함합니다.

이 프로젝트는 기존 상용 화투 게임을 대체하거나 경쟁하기 위한 제품이 아닙니다. 특정 가족이 함께 사용할 전용 환경을 직접 만들기 위한 개인 프로젝트이며, 소스 코드 공개와 불특정 다수를 위한 서비스 운영은 명확히 구분합니다.

> **GitHub 공개 범위:** 이 저장소는 화투를 공개 서비스로 배포하거나 운영하기 위한 서비스 제공 채널이 아닙니다. 개발 과정에서 작성한 소스 코드, 테스트, 문서와 재배포가 허용된 자산만 공개합니다. 실행 중인 가족용 서버, 계정, 데이터, 실제 운영 설정, 서비스 이용 권한과 운영·지원은 제공하지 않습니다.

가족용·비상업·비공개로 사용한다는 이유로 다른 사람이나 회사의 프로그램 내용과 창작물을 허가 없이 가져와도 되는 것은 아닙니다. 공개 게임 플레이 영상에서 일반적인 화투 규칙, 진행 흐름과 연출 기법을 참고하되, 구체적인 프로그램 내용·파일·고유한 표현은 추출하거나 복제하지 않는 것을 프로젝트의 기본 원칙으로 합니다.

## 프로젝트 목적

- 어르신의 접근성과 편의를 고려한 웹 기반 맞고·고스톱 게임
- 실제 가족 구성원이 함께 사용할 수 있도록 화면, 규칙과 편의 기능을 가족의 필요에 맞춰 조정하는 전용 프로젝트
- 낙장불입으로 인한 불편을 줄이는 판당 무르기와 게임머니 리필 등 편의 기능 제공
- 상용 화투 게임에서 익숙한 조작 흐름을 참고하되, 웹 환경에 맞게 화면 구성과 동작을 독립적으로 재설계

### 가족 전용 서비스라는 의미

- 개발과 운영의 일차적인 대상은 이 프로젝트를 만든 사람의 가족입니다.
- 일반 사용자를 모집하거나 누구나 가입해 이용하는 공개 서비스를 제공할 계획은 없습니다.
- 상용 게임과 비슷한 규칙이나 익숙한 조작 방식을 참고할 수는 있지만, 기존 상용 게임의 공식·비공식 대체 서비스, 복제 서비스 또는 호환 클라이언트를 만들려는 목적이 아닙니다.
- 상용 서비스와 같은 가용성, 고객 지원, 운영 인력, 장애 대응, 데이터 보전 또는 장기적인 서비스 제공을 약속하지 않습니다.
- 공개 저장소는 소스 코드를 공유하는 공간이며, 원 개발자가 운영하는 화투 서버에 대한 이용 권한을 제공하는 공간이 아닙니다.

### 공개 개발의 우선순위

- 이슈 수정과 기능 추가는 가족이 실제로 겪는 문제와 가족이 원하는 사용 경험을 해결하기 위해 진행합니다.
- 공개 저장소에 등록된 이슈나 기능 제안은 참고할 수 있지만, 접수 순서대로 처리하거나 반드시 반영하는 공통 서비스 백로그가 아닙니다.
- 여러 사용자층에 공통으로 필요한 범용 기능보다 가족의 접근성, 익숙한 규칙, 사용하는 기기와 실제 이용 방식에 맞는 변경을 우선합니다.
- 개발 방향, 기능 범위와 작업 시점은 가족의 필요에 따라 달라질 수 있으며 공개 로드맵이나 정기 배포 주기를 보장하지 않습니다.

## 웹앱 유지와 네이티브 앱 확장

Android 사용자만 고려한다면 네이티브 앱으로 제작하는 방법도 적합하지만, 실제로 함께 사용할 가족과 지인 중에는 iPhone 사용자도 있습니다. Android와 iOS 앱을 각각 개발하고 관리하려면 플랫폼별 빌드와 테스트가 필요하고, 특히 iOS 배포를 위해서는 별도의 개발자 등록 비용과 앱 배포 절차도 부담해야 합니다. 가족용 비상업 프로젝트에서는 이 비용과 관리 부담이 목적에 비해 크다고 판단했습니다.

PWA 방식은 하나의 웹 코드로 Android, iPhone과 데스크톱 브라우저에서 함께 사용할 수 있고, 앱 스토어를 거치지 않고도 홈 화면에 추가해 앱처럼 실행할 수 있습니다. 수정 사항도 서버에 배포하면 각 기기에서 바로 사용할 수 있어 가족 맞춤형 기능을 계속 개선하기에 적합합니다.

웹/PWA는 계속 독립적인 클라이언트로 유지합니다. Android는 Kotlin·Jetpack Compose, iPhone·iPad는 Swift·SwiftUI, macOS는 SwiftUI·AppKit, Windows는 C#·WinUI를 사용하는 별도 네이티브 앱으로 개발합니다. 네이티브 앱은 WebView로 웹 화면을 감싸지 않으며 각 운영체제의 UI, 저장소와 생명주기를 사용합니다. 상세 구조는 [`docs/app-platform-architecture.md`](docs/app-platform-architecture.md)를 확인하세요.

Unity 같은 범용 게임 엔진도 검토할 수 있지만, 이 프로젝트는 복잡한 3D 그래픽이나 물리 연산보다 카드 규칙, 화면 구성과 접근성 중심의 2D 웹게임입니다. 이 규모에 무거운 엔진과 런타임, 별도의 빌드 체계를 도입하는 것은 과하다고 판단하여 React, TypeScript와 웹 표준 기술로 구현했습니다.

## 제작 방식과 자산 출처

개발 과정에서 피망(Pmang), 한게임(Hangame) 등의 공개 게임 플레이 영상을 시청해 일반적인 진행 흐름, 조작 방식, 화면 배치와 연출 방향을 참고했습니다. 그러나 해당 회사 프로그램의 실행 파일·설치 디렉터리·내부 패키지·메모리·네트워크 데이터를 열거나 분석하여 자료를 취득하지 않았고, 클라이언트를 디컴파일·디스어셈블·역공학·스크래핑하지 않았습니다.

원본 프로그램이나 영상에서 코드·문구·화면·이미지·애니메이션·아이콘·글꼴·음악·음성·효과음을 추출·복사·전재·재인코딩한 형태로 프로젝트에 포함하지 않았습니다. 명시적으로 고지한 화투패, 공개 라이선스 음원과 오픈소스 의존성을 제외한 화면, 상호작용, 코드와 문구는 AI 코딩 및 이미지 생성 도구의 도움을 받아 이 프로젝트를 위해 새로 작성·설계·제작했습니다.

일반적인 화투 게임 규칙, 기능 아이디어, 조작 흐름과 UI 배치 개념을 참고하되 구체적인 소스 코드와 화면 표현은 이 프로젝트를 위해 독립적으로 작성했습니다. 다만 일반적인 아이디어·규칙과 달리 특정 서비스의 고유한 그래픽, 문구, 음원, 코드 및 창작적인 표현은 별도의 권리로 보호될 수 있으므로 이를 그대로 복제하지 않는 것을 원칙으로 합니다.

저장소에 포함된 화투패 SVG는 AI 생성물이나 NSJ의 자체 제작물이라고 주장하지 않습니다. Wikimedia Commons에 공개한 원저작자의 자산을 CC BY-SA 4.0 조건에 따라 사용하며, 저작자 표시와 동일조건변경허락 의무를 따릅니다.

### 음성 및 음악 사용 안내

공개본에는 특정 상용 게임이나 참고 영상에서 추출한 음성이 포함되어 있지 않습니다. 공개 재배포가 허용된 공유마당의 CC BY 4.0 국악 배경음, Freesound의 CC0 카드·나무 타격 효과음과 Kenney의 CC0 특수 상황 효과음을 사용합니다. 모든 효과음과 배경음은 오디오 파일로 재생하며 런타임 합성음을 만들지 않습니다. 한국어 선언은 OpenRAIL-M 공개 모델인 Supertonic 3으로 미리 생성한 AI 음성 파일을 사용하고 운영체제나 브라우저의 TTS는 사용하지 않습니다. 생성 음성임을 포함한 상세 출처와 이용 조건은 [`apps/web/public/audio/ATTRIBUTION.txt`](apps/web/public/audio/ATTRIBUTION.txt)에 기록합니다.

앞으로 별도의 음악·음성·효과음 파일을 추가할 때는 다른 콘텐츠의 분위기, 음성 톤이나 연출 방향을 참고할 수 있지만 원본 파일이나 고유한 표현을 추출·복제하지 않습니다. 직접 녹음·작곡·합성·제작하거나, 의뢰 제작 또는 정식 구입을 통해 앱 사용과 배포 권한을 확보한 자산만 사용합니다. 구입한 자산도 공개 재배포 권한이 없으면 GitHub에는 올리지 않고 `VITE_GAME_BGM_URL`을 이용한 로컬·운영 빌드 과정에서만 사용합니다.

구체적인 독자 구현 범위, 현재 화투패 출처와 향후 자산 취득·기록 원칙은 [참고 자료 및 자산 사용 정책](docs/reference-and-asset-policy.md)을 확인하세요.

> 이 설명은 프로젝트의 제작 과정과 공개 범위를 알리기 위한 것으로, 특정 서비스와의 제휴·승인 관계를 의미하거나 개별 사안에 대한 법률 의견을 제공하지 않습니다.

## 로컬 실행

Node.js, .NET 10 SDK, MySQL을 준비합니다.

```bash
npm install
```

공개 예제 파일을 복사해 `services/api/appsettings.Local.json`을 만들고 로컬 DB 연결 문자열을 입력합니다. 실제 설정 파일은 Git에서 제외됩니다.

```bash
cp services/api/appsettings.Local.example.json services/api/appsettings.Local.json
```

```json
{
  "ConnectionStrings": {
    "Hwatu": "Server=127.0.0.1;Port=3306;Database=hwatu;User ID=hwatu;Password=change-me;Character Set=utf8mb4;"
  },
  "Bootstrap": {
    "Token": ""
  }
}
```

최초 관리자 계정이 외부 방문자에게 선점되지 않도록 `Bootstrap:Token`은 기본적으로 비어 있으며, 이 상태에서는 관리자 생성이 차단됩니다. `openssl rand -hex 32`처럼 암호학적으로 안전한 방법으로 32자 이상의 일회성 토큰을 만들고, Git에서 제외되는 `services/api/appsettings.Local.json`에만 입력합니다. 관리자 생성을 마치면 토큰 값을 다시 비우고 서버를 재시작합니다. 운영 환경에서는 추적되는 설정 파일 대신 환경 변수 `Bootstrap__Token` 또는 별도의 비밀 저장소를 사용하세요.

DB와 사용자를 만든 뒤 서버와 프런트엔드 개발 서버를 각각 실행합니다. 서버 시작 시 `services/api/database/schema.sql`을 적용합니다.

```bash
npm run dev:server
npm run dev
```

브라우저에서 `http://127.0.0.1:5234`에 접속한 뒤 서버에 설정한 일회성 토큰으로 최초 관리자 계정을 등록합니다. 새 비밀번호는 15자 이상이어야 합니다.

### 직접 구동하려는 사용자의 책임

이 저장소는 코드를 제공할 뿐, 누구나 바로 접속할 수 있는 공용 인스턴스나 관리형 호스팅을 제공하지 않습니다. 가족 외의 사람이 이 코드를 가져가 사용하려면 자기 환경에 필요한 조건을 직접 마련하고 운영해야 합니다.

- Node.js, .NET 10 SDK와 MySQL을 직접 설치하고 호환되는 실행 환경을 구성해야 합니다.
- 데이터베이스, 계정, 연결 문자열과 애플리케이션 설정을 직접 생성하고 안전하게 보관해야 합니다.
- 외부에서 접속할 서버를 운영한다면 도메인, DNS, HTTPS 인증서, 방화벽, 리버스 프록시와 네트워크 보안을 직접 준비해야 합니다.
- 배포, 프로세스 관리, 로그 확인, 모니터링, 백업, 복구, 업데이트와 장애 대응을 직접 수행해야 합니다.
- 사용자 계정과 저장 데이터에 관한 접근 통제, 개인정보 보호, 관련 법령 및 운영 지역의 요구사항을 직접 검토해야 합니다.
- 추가한 이미지, 음원, 글꼴과 다른 제3자 자산의 이용·배포 권한을 직접 확인해야 합니다.
- 자기 환경에서 발생한 비용, 데이터 손실, 보안 문제, 호환성 문제와 서비스 중단은 해당 환경을 구성하고 운영한 사람이 관리해야 합니다.

프로젝트에 포함된 예시 설정과 배포 파일은 이 프로젝트의 환경을 설명하기 위한 참고 자료입니다. 다른 환경에서 그대로 동작하거나 안전하다고 보장하지 않으므로, 각 사용자가 자기 인프라와 보안 요구사항에 맞게 검토하고 수정해야 합니다.

```bash
npm run check:public
```

이 명령이 실패하면 출력된 항목을 해결하기 전에는 푸시하지 않습니다. 특히 현재 브랜치 이력에 비공개 운영 파일이 연결돼 있다는 오류가 나오면, 검증된 `origin/main`을 부모로 하는 공개 커밋에 현재 정제된 파일만 반영해야 합니다.

개별 검증 명령은 다음과 같습니다.

```bash
npm test
npm run typecheck
npm run build
dotnet build services/api/Hwatu.Server.csproj
```

Vite 빌드 결과는 `services/api/wwwroot/`에 생성되며 저장소에는 포함하지 않습니다. 루트 npm 명령은 `apps/web/` 워크스페이스에서 기존 웹 테스트와 빌드를 실행합니다. 네이티브 앱은 각 `apps/android/`, `apps/ios/`, `apps/macos/`, `apps/windows/`에서 독립적으로 빌드합니다.

GitHub 공개용 운영 설정 예제는 [`deploy/examples/`](deploy/examples/)에 있습니다. 실제 도메인, 서버 사용자, 설치 경로와 인증서 경로를 반영한 운영 설정은 `deploy/`의 예제 외 경로에 별도로 만들며 Git에서 제외됩니다.
